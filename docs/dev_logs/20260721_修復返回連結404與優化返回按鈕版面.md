## 修改內容說明

### 1. 返回連結 404 問題修復 (`templates/report_template.html`)
- **問題分析**：戰報目錄重構分層存檔至 `reports/YYYY/MM/YYYY-MM-DD.html` 後，原模板中的相對路徑 `../index.html` 只能退回 `reports/YYYY/`，導致點擊「返回戰報列表」時出現 404 找不到檔案。
- **修復方案**：將模板中的 href 改為 `../../../index.html`，以正確向上退回三層抵達專案根目錄。

### 2. 返回首頁按鈕版面優化與絕對定位 (`templates/report_template.html`)
- **定位調整**：將 `.back-home` 樣式設為 `position: absolute; top: 0.5rem; left: 0;`。因為 `header` 本身已具備 `position: relative`，返回連結得以在不佔用獨立行空間的情況下，置於頁面左上角，使頂部大標題排版更為對稱美觀。
- **視覺美化**：
  - 新增 `background: rgba(255, 255, 255, 0.03);` 與 `border: 1px solid var(--card-border);`，並增加 `padding: 0.4rem 0.8rem;` 及圓角，使其呈現高級的 Chip 按鈕質感。
  - 將 SVG 寬高微調為 `16px`。
- **行動端防重疊相容**：
  - 增加 `@media (max-width: 576px)` 媒體查詢，在窄螢幕行動裝置下自動將按鈕改回靜態流定位 (`position: static`)，防止按鈕與置中的標題文字發生重疊碰撞。

### 3. 進度與戰報更新
- 重新生成昨日賽事戰報 `reports/2026/07/2026-07-20.html`，確認返回連結 href 正常變更、樣式美化成功，且首頁索引更新無誤。
