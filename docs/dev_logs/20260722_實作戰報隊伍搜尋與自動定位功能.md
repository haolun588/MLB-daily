## 💡 改動背景與目的
隨著單日賽事數量較多（最高可達 15~16 場比賽），使用者在開啟每日戰報時需要向下滾動多次才能找到特定關注的球隊賽事。為了提升檢索效率與使用者體驗，新增深色系搜尋輸入框，支援輸入中/英文隊名與縮寫，並在搜尋後自動平滑捲動定位與發光提示。

---

## 🛠️ 詳細變更內容

### 1. 搜尋 UI 與樣式設計 (`templates/report_template.html`)
- 新增 `.search-section` 玻璃擬態深色搜尋框，配備搜尋放大鏡 Icon 與藍色漸變搜尋按鈕。
- 新增 `@keyframes highlightPulse` 與 `.game-card.highlight-card` 特效：
  - 當搜尋定位成功時，目標比賽卡片會觸發柔和藍色外發光（`box-shadow: 0 0 25px rgba(59, 130, 246, 0.6)`）與微幅放大（`scale(1.02)`）提示動畫。
  - 動畫於 2.5 秒後自動結束並還原原樣。

### 2. 卡片 DOM 屬性注入 (`fetch_mlb.py`)
- 在 `generate_report()` 生成 HTML 卡片時，將雙方隊伍的關鍵字資訊寫入 `<div class="game-card">` 的 `data-teams` 屬性。
- `data-teams` 包含了：
  - **隊伍縮寫**（如 `LAD`, `NYY`）
  - **中文官方名稱**（如 `洛杉磯道奇`, `紐約洋基`）
  - **英文官方名稱**（如 `Los Angeles Dodgers`, `New York Yankees`）

### 3. JavaScript 前端檢索與定位邏輯 (`templates/report_template.html`)
- 監聽搜尋輸入框 `Enter` 鍵與「搜尋」按鈕點擊事件。
- 不分大小寫搜尋匹配第一個符合條件的 `.game-card[data-teams]`。
- 使用 `scrollIntoView({ behavior: 'smooth', block: 'center' })` 將目標卡片置中捲動呈現。
- 自動重置並觸發 `.highlight-card` 動畫。
- 無匹配結果時，於搜尋框下方呈現提醒訊息。

### 4. 專案進度更新 (`docs/progress.md`)
- 標記「階段十一：搜尋與自動定位跳轉」全數勾選完成。
