# 開發守則 (Developer Guidelines)

本專案在開發過程中必須嚴格遵守以下守則。**每次開工前，AI Agent 必須閱讀並確認本守則內的所有規則。**

---

## 🚫 核心限制與開發順序

1. **先規劃、後實作**：
   - 在任何程式碼變更前，必須先與使用者討論，並確認 `docs/` 下的專案架構與進度表均已更新。
2. **模組化開發 (One Feature at a Time)**：
   - 每次只開發一個模組或功能。禁止同時修改多個無關的模組。
   - 每個模組完成後，必須進行手動或自動驗證，確認無誤後才能進入下一個模組。
3. **UI 優先 (UI-First Approach)**：
   - **必須先做好 HTML/CSS 的靜態頁面架構與視覺設計**。
   - 只有在使用者親自開啟網頁瀏覽、確認視覺效果（深色模式、響應式佈局、美感）滿意並批准後，才能開始撰寫 Python 抓取腳本與自動化功能。

---

## ⚙️ 系統設計守則

4. **台北時間中午 12:00 開工檢查邏輯**：
   - 每日排程觸發時間為台北時間中午 12:00 (12:00 PM)。
   - **開工前置檢查**：腳本啟動後，必須先透過 API 檢查昨日的所有比賽狀態。
   - **延遲等待機制**：
     - 若有任何一場比賽尚未結束（例如仍處於進行中或延遲狀態），腳本必須**暫停並休眠 30 分鐘**，之後重新發起檢查。
     - 只有在**確定所有比賽均已打完 (Final) 或已延期 (Postponed)** 後，腳本才能正式開始擷取資料並生成網頁。
5. **程式設計原則**：
   - 儘量使用 Python 原生標準函式庫 (如 `urllib.request` 與 `json`)，避免在 GitHub Actions 環境中安裝不必要的第三方依賴套件，確保程式輕量化與執行穩定度。
6. **通知與傳送**：
   - 戰報網頁生成並成功推送至 GitHub 後，必須發送 Discord Webhook 通知使用者。通知內必須附帶可直接點擊打開的 GitHub Pages 連結。

---

## 🚀 開工與收工工作流 (Session Commands Workflow)

7. **「開工」指令流程**：
   - 當使用者在任何對話說 **「開工」** 時，AI Agent 必須：
     1. 檢查開發行為守則（`docs/dev_rules.md`）與當前進度表（`docs/progress.md`）。
     2. 與使用者確認當次預計進行的工作內容。
     3. 等待使用者確定同意後，方可開始進行程式碼修改或開發。
8. **「收工」指令流程**：
   - 當使用者在任何對話說 **「收工」** 時，代表該次工作結束，AI Agent 必須：
     1. 根據該次工作實際變更內容，提供 push 至 GitHub 的 `summary` (標題) 與 `description` (詳細描述)。
     2. 於 `docs/dev_logs/` 資料夾下生成一份該次開發日誌。
     3. 日誌命名規則為：`YYYYMMDD_標題`（例如 `20260720_優化某某功能.md`）。
     4. 日誌內容簡述本次新增與改動的項目。

## 📊 戰報數據與呈現規格 (Report Data & Presentation Specs)

### 1. 打者數據規格 (Batters Specs)
- **基本打擊數據**：打數 (AB)、得分 (R)、安打 (H)、打點 (RBI)、保送 (BB)、被三振 (SO)。
- **長打 (Extra-base hits)**：二壘安打 (2B)、三壘安打 (3B)、全壘打 (HR)。
- **速度指標**：盜壘 (SB)。
- **累計數據標記格式**：
  - 必須標記長打與盜壘在「該場比賽的次數」以及「該次數對應的賽季累計總數」。
  - 格式範例：`HR: 1(9)` 代表該場比賽有 1 支全壘打，且該全壘打為賽季第 9 轟。
  - 複數次數範例：`HR: 2(10)(11)` 代表該場比賽有 2 支全壘打，分別為賽季第 10 轟與第 11 轟。
  - 盜壘範例：`SB: 1(15)` 代表該場比賽有 1 次盜壘成功，且為賽季第 15 次盜壘成功。

### 2. 投手數據規格 (Pitchers Specs)
- **投球數據**：投球局數 (IP)、被安打 (H)、失分 (R)、自責分 (ER)、保送 (BB)、三振 (SO)、總投球數/好球數 (Pitches/Strikes)。
- **移除項目**：**移除**原本規劃的「賽季防禦率 (ERA)」，專注於當場比賽的投球內容。
